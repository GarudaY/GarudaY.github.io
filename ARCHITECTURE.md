# Architecture

## Продуктовая структура

Информационная архитектура держит первый уровень коротким:

- Главная
- Про нас
- Курсы
- Події
- Новости
- Поддержать
- Контакты

Юридические страницы находятся в footer: `Impressum`, `Datenschutzerklärung`, `Cookie-Einstellungen`.

Основные сценарии:

- украинцы в Германии быстро находят курсы, события и контакты;
- родители видят возраст, место, цену, расписание и статус набора;
- участники событий видят дату, время, место, цену, остаток мест и отдельную онлайн-регистрацию;
- доноры понимают назначение помощи и видят способы поддержки;
- партнеры находят миссию, команду, проекты и официальные контакты;
- волонтеры получают путь к контакту.

### Аудитории и приоритетные задачи

| Аудитория           | Главная задача                                             | Короткий путь                              |
| ------------------- | ---------------------------------------------------------- | ------------------------------------------ |
| Украинцы в Германии | Найти обучение, встречу или поддержку                      | Главная → курсы / события / контакты       |
| Родители            | Проверить возраст, расписание, преподавателя, цену и места | Курсы → карточка → sticky summary → запрос |
| Участники событий   | Увидеть дату, время, место, цену и регистрацию             | События → детальная страница → форма       |
| Доноры              | Понять назначение средств и доступные способы              | Главная / Header → поддержать              |
| Партнеры            | Проверить миссию, команду, проекты и контакты              | Про нас → контакты                         |
| Волонтеры           | Понять направления помощи и связаться                      | Про нас → контакты                         |

### Ответственность страниц

- Главная: идентичность, актуальные действия, курсы, ближайшие события, доверие, пожертвования и партнеры.
- Про нас: миссия, ценности, формат работы, команда, правление, преподаватели, партнеры и CTA сотрудничества.
- Курсы: короткие полезные фильтры; детали курса содержат программу, результаты, материалы, преподавателей и summary для записи.
- События: ближайшие и архив; детали содержат все данные для решения об участии, остаток мест, структурированную форму регистрации и связанные материалы.
- Новости: объявления и отчеты со связями на курсы и события.
- Поддержать: назначение помощи и явно демонстрационные способы пожертвования без платежного backend.
- Контакты: централизованные данные, полезный блок планирования визита и demo-форма для общих запросов. Регистрация на событие вынесена в отдельный flow и не смешивается с общей почтой.
- Legal: только шаблоны с предупреждением, без претензии на юридическую корректность.

Навигация ограничена семью пунктами первого уровня. Профили людей и юридические страницы доступны контекстно, а не перегружают Header.

## Маршруты

Внутренние маршруты держатся стабильными:

```text
/: redirect to /uk
/:locale
/:locale/about
/:locale/courses
/:locale/courses/:slug
/:locale/people/:slug
/:locale/events
/:locale/events/:slug
/:locale/news
/:locale/news/:slug
/:locale/donate
/:locale/contact
/:locale/impressum
/:locale/privacy
/:locale/cookies
```

Публичные немецкие сегменты локализованы через `src/proxy.ts`, как требует Next.js 16 для проекта с `src/app`: например `/de/kurse` переписывается во внутренний маршрут `/de/courses`. Прямой внутренний немецкий URL перенаправляется на канонический локализованный вариант, чтобы не создавать дубликаты.

## Слои

- `src/content/mock/*`: временный источник данных.
- `src/data/content.ts`: единый data access layer.
- `src/types/content.ts`: строгие модели.
- `src/components/*`: UI и content-компоненты.
- `src/app/*`: маршруты, metadata, structured data.

Страницы не импортируют mock-массивы напрямую. При подключении CMS нужно заменить реализацию функций `getCourses`, `getEvents`, `getNewsArticles`, `getPeople`, `getDonationSettings` и т.д.

## Локализация

Основные локали: `uk`, `de`. Архитектура позволяет добавить `en` и `ru`:

- локализованные поля данных: `LocalizedString`, `LocalizedList`;
- словари интерфейса: `src/i18n/dictionaries.ts`;
- fallback на украинский: `t()` и `tList()`;
- локализованные URL: `src/i18n/routing.ts`;
- `hreflang` в metadata helper.

Даты и время форматируются для `Europe/Berlin`, заданной централизованно в `src/config/site.ts`, поэтому серверный timezone хостинга не сдвигает время событий.

Украинский используется как fallback для отсутствующего поля. Текущий набор локалей типизирован как `uk | de`; добавление `en` или `ru` требует расширить конфигурацию, словари и локализованные значения, но не менять UI-контракты.

## SEO

Реализовано:

- Next Metadata API;
- title template;
- descriptions;
- Open Graph;
- canonical;
- alternates / hreflang;
- `sitemap.xml`;
- `robots.txt`;
- Organization, Event, Article, Course и BreadcrumbList JSON-LD там, где это уместно.

Noindex-страницы (юридические placeholders и demo-профили) не включаются в sitemap. Sitemap содержит языковые alternates для индексируемых маршрутов.

## Accessibility

Основа:

- semantic HTML;
- один `h1` на страницу;
- skip link;
- видимые focus states;
- keyboard-friendly header/mobile nav;
- labels для формы;
- `aria-current`, `aria-expanded`;
- не используется clickable div;
- контентные изображения имеют локализованные alt-тексты;
- `prefers-reduced-motion`.

## Производительность

По умолчанию используются Server Components. Client Components ограничены интерактивными зонами: header/mobile nav, language switcher, course filters, горизонтальные профили, contact form и event registration form. Нет аналитики, tracking, animation library или тяжелой UI-библиотеки.
