# Ukrainian Verein Website Prototype

Современный frontend-прототип сайта украинского Verein в Германии. Проект создан как CMS-ready основа: страницы работают на локальных mock-данных, но UI получает данные через `src/data/content.ts`, поэтому позднее источник можно заменить на CMS API без переписывания страниц.

## Стек

- Next.js `16.2.10`, App Router
- React `19.2.4`
- TypeScript
- Tailwind CSS `4`
- ESLint `9`
- Prettier
- Lucide React для иконок
- Zod используется для серверной валидации заявок и будущих CMS/API payloads

## Запуск

```bash
npm install
npm run dev
```

По умолчанию сайт открывается на `/uk`. Немецкая версия доступна через `/de`, с локализованными публичными сегментами вроде `/de/kurse`.

Ключевые публичные разделы: главная, о Verein, курсы, профиль преподавателя, мероприятия, новости, пожертвования, контакты и юридические demo-шаблоны. Детальные страницы курсов, событий и новостей генерируются из контентного слоя.

## Проверки

```bash
npm run typecheck
npm run lint
npm run build
```

## Переменные окружения

```text
NEXT_PUBLIC_SITE_URL=https://example-verein.de
```

`https://example-verein.de` является mock-доменом. Перед production его нужно заменить на настоящий домен Verein.

## Структура

```text
src/app/                 App Router routes, metadata, sitemap, robots
src/components/layout/   Header, footer, logo, language switcher
src/components/ui/       Базовая дизайн-система
src/components/content/  Карточки и контентные блоки
src/components/events/   Отдельный flow регистрации на события
src/content/mock/        Демонстрационные данные
src/data/content.ts      Data access layer
src/i18n/                Локали, словари, маршрутизация
src/types/               Контентные TypeScript-модели
src/config/              Домен и глобальные настройки
public/images/generated/ Оптимизированные demo-фотографии
public/images/partners/  Векторные demo-логотипы партнеров
```

## Demo-контент

Все люди, адреса, платежные реквизиты, партнеры и юридические данные являются вымышленными. Фотографии сгенерированы специально для прототипа и оптимизированы в WebP; реальные изображения, реквизиты и юридические тексты нужно добавить перед публикацией.

Регистрация на событие и контактная форма работают через локальные Route Handlers. Данные атомарно сохраняются в исключённый из Git файл `.data/operations.json`; поддерживаются duplicate-защита, остаток мест, waitlist, приватная отмена, статусы обращений и CSV-экспорт. Локальная панель доступна на `/uk/admin` и `/de/admin` только через localhost. Подробности и ограничения описаны в `OPERATIONS.md`.

Почтовая доставка не имитируется: после регистрации пользователь получает номер и приватную ссылку прямо в интерфейсе. Для serverless или многосерверного production потребуется внешняя транзакционная база и почтовый провайдер.

## Замена логотипа и изображений

- Временный знак находится в `src/components/layout/SiteLogo.tsx`. Чтобы заменить его централизованно, измените только этот компонент; Header и Footer используют его повторно.
- Demo-фотографии находятся в `public/images/generated/`, партнерские знаки — в `public/images/partners/`. Путь и локализованный `alt` задаются в `src/content/mock/*` через `ImageAsset`.
- После подключения CMS `ContentImage` продолжит принимать тот же контракт. Для удаленных изображений нужно добавить точный `remotePatterns` в `next.config.ts` и передавать размеры либо `fill` с устойчивым aspect ratio.

## Переход на CMS

Рекомендуемый первый вариант — Sanity, с оговорками по ролям и тарифу, описанными в `CMS_PLAN.md`. Mock-файлы не импортируются страницами напрямую: адаптер в `src/data/content.ts` можно заменить запросами CMS, сохранив публичные функции и компоненты.

## Публичный demo-хостинг

Cloudflare/OpenNext-сборка и готовый bundled Worker создаются командой:

```bash
npm run bundle:sites
```

Для бесплатного публичного preview устанавливается runtime-переменная `HOSTED_DEMO=true`. В этом режиме заявки и обращения хранятся в памяти Worker: весь flow работает, но demo-данные могут сброситься при перезапуске изолята. Локальный режим без этой переменной по-прежнему использует `.data/operations.json`. Для production с реальными заявками нужна постоянная транзакционная база данных.
