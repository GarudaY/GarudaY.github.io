# Local operations

Проект умеет принимать регистрации и контактные обращения без почтового сервиса, CMS или внешней базы данных.

## Что работает

- серверная Zod-валидация и ограничение размера запросов;
- защита same-origin, honeypot и локальный rate limit;
- атомарное хранение в `.data/operations.json`;
- уникальность активной регистрации по `event + email`;
- подтверждение по доступным местам и автоматический waitlist;
- продвижение первой подходящей заявки из очереди после отмены;
- приватная ссылка просмотра и отмены;
- контактная очередь со статусами `new`, `in_progress`, `resolved`;
- локальная админка `/uk/admin` или `/de/admin`;
- CSV-экспорт регистраций и обращений.

Админка и экспорт отвечают только при открытии через `localhost` или `127.0.0.1`. На публичном hostname API возвращает `403`, а страница — `404`.

## Хранение и резервные копии

По умолчанию файл создаётся в `.data/` и исключён из Git. Путь можно изменить через `LOCAL_DATA_DIR`. Для реального self-hosted запуска каталог должен находиться на постоянном диске и регулярно копироваться.

Отменённые заявки и закрытые обращения удаляются при очередной записи через `LOCAL_DATA_RETENTION_DAYS` дней, по умолчанию через 30. Активные заявки и незакрытые обращения автоматически не удаляются.

Файловое хранилище подходит для одного локального или небольшого self-hosted Node-процесса. Оно не подходит для serverless-хостинга с временной файловой системой или нескольких параллельных серверов — там потребуется внешняя транзакционная база.

## Проверка

При запущенном `npm run dev` выполнить:

```bash
npm run test:operations
```

Скрипт временно резервирует текущий файл, проверяет регистрацию, duplicate-защиту, waitlist, автоматическое продвижение, отмену, контактную очередь, админский API и CSV, после чего восстанавливает исходные данные.

## Что намеренно не имитируется

- отправка email и SMS;
- настоящие платежи;
- юридически утверждённые тексты согласий;
- публичная удалённая админка и управление пользователями.

Для этих функций нужны реальные аккаунты, реквизиты, политика доступа или внешний сервис.
